git checkout -- <file name>
git checkout -- <directory name>
  cancel any changed of File or directory
  ワークツリーにて内容を変更
  (addはまだしていない)
  git status
  git checkout -- <file name>
  ワークツリー上の変更が戻される


git checkout -- .
  全変更を取り消す
  --をつけているのは，ブランチ名とファイル名が被った時にどちらを指しているのか
  Gitが分からなくなることを避けるため．
  ピリオドは任意のファイル全てという意味．

git reset HEAD <file name>
or
git reser HEAD .
  ステージした変更を取り消す
  Cancel any changes that have made to the stage.
  make to :する，なる
  ※このコマンドは，ステージに上げた変更を取り消すだけなので，ワークツリーの変更には影響が無い！
  理解．

git commit --amend
  直前のコミットをやり直す
  ※pushしたものは変更NG
  これを実行すると，今のstaging areaをもとにコミットをやり直す，という意味になる
  つまり，失敗したら
  ワークツリーでそのファイルを変更
  git add
  それで変更したファイルをステージングエリアにのせてから，このコマンドを実行する．


git remote add <remote name> <remote URL>
  リモートリポジトリを新規追加する．
  これを実行すると，今後はTutorialという名前でGithubリポジトリにアップしたり取得出来たりする．

  example)
    git remote add tutorial https.//....
  　-> MY: Registered URL ,that is shortcut, for remote repository name as 'tutorial' .
    -> -> Resistor the URL of the remote repository with a shortcut named tutorial
    Tutorial という名前のショートカットで，リモートリポジトリのURLを登録する


git remote
  設定しているリモートリポジトリの情報を確認する事ができる
  現時点で登録されているショートカットは
  origin
  bak
  の2つ
　-vで各リポジトリのショートカットのURLが表示される


git push <remote repository shortcht name> <branch>
  ex) git remote origin master
      git remote -u bak master

git fetch <remote name>
  retrieving information from a remote repository
  練習
  まず，Githubの方でhome.htmlを作成して，”リモートリポジトリ内”でコミット
　そのあと
  git fetch origin
  ここで取得した情報はどこに保存されているか確認
  git branch -a
  今いるブランチに※がつく

git checkout remotes/origin/master
  remotes/origin/masterに，
  自分のワークツリーの内容を切り替えるという事を行っている
　もとのブランチに戻る場合は
  git checkout master

  最後に，取得してきたリモートリポジトリの情報を，
  ワークツリーに取り込む．

git pull <remote name> <branch name>
  -> merged information fetched from the remote repository
  リモートから情報を取得して，マージまで行う作業を一括で行いたい時に使用するコマンド
      git pull origin master
      =
      git fetch origin master
      git merge origin/master

git remote show <remote name>
git remote show origin
  リモートリポジトリの情報を詳細に知ることができる


git branch <branch name>
  ブランチを新規作成するコマンド．
  MY: The command, create new branch.
  Dp: Command to create a new branch.
  このコマンドはブランチを作成するだけで，ブランチの切替までは行わない．
  This command only can do that, not switch branch pointer.
  This command only creates a branch, but does not switch branches.
  ex)git branch feature


git rebase <branch name>
  ブランチの基点となるコミットを別のコミットに移動する．
