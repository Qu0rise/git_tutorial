バージョン管理する必要が無いファイル
  ・自動生成されるファイル
  ・個人情報を含んだファイル

--をつけているのは，ブランチ名とファイル名が被った時にどちらを指しているのか
Gitが分からなくなることを避けるため．


HEADは，今自分がいるコミットの最新のブランチを指す！！！

つまり，
git reset HEADで，最新のコミットの内容で
リセットするよ，ステージの内容を上書きするよ
ということを指す．



リモートリポジトリは，複数持つ事ができる
  複数持つ理由としては，会社で共有するもの以外に
  自分自身で持つものも存在する．


リモートリポジトリにある情報を確認したい場合には，
remote fetchコマンドを使って情報を収集し
ワークツリーに情報を持ってくるために
git merge を行う．


ブランチ：並行して複数機能を開発したい時に使用するのがブランチ．
ブランチを理解する上で，Gitのデータの持ち方を理解する事は非常に重要．
